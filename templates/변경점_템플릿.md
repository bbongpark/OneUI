# 변경점 열 작성 템플릿 (초안)

> [회사 작업] 실제 취합 엑셀의 변경점 열 실물을 보고 이 템플릿을 다듬을 것.
> PL 페르소나가 이 템플릿 준수 여부를 검사한다 (prompts/persona-pl.md 검사 1 참조).

## 필수 구성 요소

변경점 열에는 아래 4가지가 모두 담겨야 한다. 임원이 이 열만 읽고 변경의 본질을 파악할 수 있어야 한다.

```
[변경 전] <기존 동작을 한 문장으로>
[변경 후] <바뀌는 동작을 한 문장으로>
사유: <왜 바꾸는가 — VOC 건수, 경쟁 대응, 전략 과제 등 정량 근거 우선>
영향: <영향 범위 — 모듈, 사용자군, 타 부서 의존성>
```

## 작성 예시

```
[변경 전] 잠금화면 위젯은 고정 2종만 제공
[변경 후] 위젯 스택 및 크기 조절 지원, 서드파티 위젯 허용
사유: VOC 1,240건 (2026.1~5), 경쟁사 대비 커스터마이징 열세
영향: 잠금화면 모듈, 전 모델. 서드파티 위젯 정책은 보안팀 협의 필요
```

## 미준수로 판정되는 경우

- 변경 전/후 구분 없이 결과만 서술 ("위젯 개선")
- 사유가 정성적 주장뿐 ("고객이 원함" — 수치 없음)
- 영향 범위 누락
- "-", "TBD", "추후 업데이트" 등 자리채움 값
