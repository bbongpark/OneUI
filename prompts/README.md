# prompts/ — 페르소나 프롬프트

Claude CLI 워커에 주입되는 프롬프트 파일들. 대시보드의 페르소나 편집기가 이 파일들을 읽고 쓴다.

**리뷰 회의의 정체**: SW 담당 부사장급 임원 + 각 부문(경험기획·UX·개발·CXI·PL) 임원이 "이 Feature를 해당 버전에 진행할 것인가"를 결정하는 go/no-go 의사결정 회의. 모든 페르소나는 이 결정을 준비하는 부문별 권고안 초안을 만든다. 따라서 리뷰 페르소나 4종은 등급(P0/P1/P2 = 임원 대면 판단 필요도)과 별도로 진행 권고(go/conditional_go/defer/no_go)를 낸다.

## 구성

| 파일 | 역할 | 실행 시점 |
|---|---|---|
| `_common.md` | 공통 규칙 (비대화형, JSON 출력, needs_human, P0/P1/P2 정의) — 모든 호출 앞에 자동 주입 | 항상 |
| `persona-experience-planning.md` | 경험기획: 사용자 가치·전략·차별화·시나리오 완결성 | 리뷰 단계 |
| `persona-ux.md` | UX: 사용성·디자인 일관성·접근성·폼팩터·습관 파괴 | 리뷰 단계 |
| `persona-dev.md` | 개발: 실현성·성능·안정성·보안·의존성·일정 현실성 | 리뷰 단계 |
| `persona-cxi.md` | CXI: VOC 정합성·불만 해소·커뮤니티 반응·지원 비용 | 리뷰 단계 |
| `persona-synthesis.md` | 종합: 4개 결과를 최종 P0/P1/P2로 판정, 의견 분산 감지 | 리뷰 단계 (개별 4종 완료 후) |
| `persona-pl.md` | PL: 문서 완결성 + 발표 준비 + 일정 리스크 | PL 검사 단계 |
| `persona-sw-director.md` | SW담당 임원: 배정된 안건의 예상 go/no-go + 확신도 + 예상 질문. **판단 성향 섹션은 회사에서 실제 회의록 기반으로 교체** | 일정 배정 후 (④-2 예상 판정) |

## 호출 규약

- 실행 프롬프트 = `_common.md` + 해당 페르소나 파일 + 배치 데이터(JSON)
- 개별 4종은 병렬 개념이지만 워커 큐에서 순차 실행됨 (배치 10~20건 단위)
- 종합은 4종 결과가 모두 모인 Feature만 처리
- 모든 출력은 JSON 파싱 후 `data/<버전>/`에 저장됨 — 스키마를 바꾸면 파서(서버 쪽)도 함께 바꿔야 한다

## 수정 시 주의

- 출력 스키마의 필드명은 대시보드·파서와 계약이다. 문구(관점, 가이드)는 자유롭게 다듬되 스키마 변경은 신중하게.
- 회사 환경에서 실제 데이터로 돌려보며 판정 기준(P0 남발/부족)을 튜닝하는 것이 1차 과제.
