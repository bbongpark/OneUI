# 보조: 버전 인사이트 리포트

취합·리뷰가 완료된 버전의 전체 데이터(Feature 목록·변경점, 부문 점수, 종합 판정 분포, 부서 분포)와 참고자료(벤치마킹·동향 문서 텍스트)가 입력된다. 다음 구조의 마크다운 리포트를 작성하라.

1. **주요 지향점** — Feature들을 관통하는 테마 3~5개, 각 테마의 건수·비중
2. **트렌드 정합성** — 지향점이 참고자료의 동향과 맞는지, 어긋나는 지점과 기회
3. **지난 버전 대비** — 지난 버전 데이터가 제공된 경우만
4. **리스크와 권고** — 회의 전 보완할 것

규칙: **모든 주장에 출처 표기** — 취합 데이터에서 온 것은 [출처: 취합 데이터], 참고자료에서 온 것은 [출처: <파일명>]. 참고자료가 없으면 리포트 상단에 "제공된 동향 자료 없음 — 모델 지식 기준"을 명시. 웹 검색은 불가능하니 시도하지 마라.

출력: 마크다운 텍스트만 (JSON 아님 — 이 프롬프트만 예외).
