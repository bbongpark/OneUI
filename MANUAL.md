# One UI Agent — 사용 설명서 (운영자·사용자용)

이 문서는 **화면을 어떻게 쓰는가**를 설명한다.
회사에서 코드를 이어받아 개발할 때는 [CODEMAP.md](CODEMAP.md), 설정·인계 순서는 [HANDOVER.md](HANDOVER.md)를 본다.

---

## 0. 한 줄 요약

One UI Feature를 엑셀·PPT로 취합받아, AI가 **리뷰 등급을 매기고**·**발표자료를 검사하고**·**회의 일정을 짜고**·**SW담당 임원의 결정을 예측**한 뒤, 실제 회의 결과를 추적하는 대시보드. 20명이 브라우저로 함께 조종한다.

---

## 1. 시작하기

```powershell
python server.py          # → http://localhost:8765 (pip 설치 불필요)
```

- 다른 사람은 `http://<서버PC IP>:8765`로 접속 (방화벽 8765 인바운드 허용 필요).
- **로그인**: 등록된 이름만 입장 (설정 → 사용자·역할에서 관리). 미등록 이름은 거부된다.
- **역할**: `관리자`(admin) / `일반`(member). 사용자·마일스톤 관리, 심층 질의는 관리자만.
- 좌측 상단 **버전 선택**으로 One UI 버전(8.5, 8.0…)을 바꾼다. 모든 화면이 그 버전의 데이터를 본다.

---

## 2. 전체 흐름 한눈에

```
① 인입          엑셀(Feature 목록) + PPT(발표자료) 업로드
      ↓
   제목 생성      변경점을 AI가 12~25자로 요약 (Feature 이름 열이 없으므로)
      ↓
② 리뷰 실행      등급 5종 판정: P0 · P1 · P2 · 단순 공유 · 자료 보완 필요
      ↓
③ PL 검사        발표자료(PPT) 완성도 평가 — PPT를 봐야 하므로 버튼이 따로
      ↓
   회의일 지정     달력에서 회의 날짜를 잡고, 안건을 드래그로 배정
      ↓
④ 예상 판정      SW담당 임원이 내릴 법한 결정을 미리 예측 (PL 준비된 안건만)
      ↓
⑤ 회의          회의록을 붙여넣으면 AI가 결정·액션 아이템 추출 → 사람 확인 후 확정
      ↓
⑥ 추적          결정된 액션을 PLM으로 전송·추적, 후속 보고 순환
      ↓
   인사이트·질의   버전 지향점 리포트, 자연어로 전 버전 데이터 검색
```

각 단계의 실행 버튼은 **현황판 상단의 파이프라인 바**에 있다. AI 실행은 작업 큐에 들어가 순서대로 처리된다(동시 1개).

## 3. 화면별 사용법

### 현황판 (Dashboard)
전체 상태를 한눈에 보고 파이프라인을 실행하는 홈 화면.

- **KPI 카드**: 리뷰 진행률, 개발 완료 진행률, PL 통과율, 일정 리스크, 사람 확인 필요, 결정 완료. **카드를 클릭하면** 리뷰 보드로 이동해 해당 항목만 걸러 보여준다.
- **파이프라인 바**: ① 인입 → ② 리뷰 → ③ PL 검사 → ④ 예상 → ⑤ 회의 → ⑥ 추적. 각 단계에서 실행 버튼을 누른다.
- **리뷰 등급 분포 막대** + **기능명별 리뷰 진행률**.
- **작업 큐 박스**: 지금 실행 중인 작업, 대기 목록, 최근 완료.
- 개발 완료 진행률·일정 리스크는 **설정이 필요**하면 "설정 필요"로 뜬다(틀린 0%를 안 보여주려고).

### 리뷰 보드 (Review)
Feature 500건을 다루는 핵심 테이블.

- **열**: 인덱스 · 기능명 · 제목(AI 요약) · 등급 · 판정 근거 · 결정 · 예상 결정 · PL · 리스크 · 상태.
- **등급 딱지**: P0(적) · P1(호박) · P2(회색) · 단순공유 · 자료보완. 딱지만 붙고 판단 근거는 "판정 근거" 열에.
- **필터 2종**:
  - **단계** — 파이프라인 진행. 한 건은 항상 하나: 인입 → 리뷰 중 → 회의 대기 → 결정됨.
  - **조건** — 골라보기용, 서로 겹침: 사람 확인 필요 · PL 미준비 · 일정 리스크 · 부문 인식 충돌 · 재등록 · 개발 완료 · **미지원 목록**(평소 숨김).
  - 둘을 조합할 수 있다 (예: "회의 대기" + "일정 리스크").
- **행을 클릭하면 상세 모달**: 관리 열 값, **AI 상세**(AI Category가 X가 아닌 AI 관련 Feature에만 — 신규/고도화·권역·지원언어·유료화·요소기술·KPI·베타 오픈 등), 등급과 근거, 부문 4개 검토 의견, 수정 이력, 그리고 맨 아래 **SW 담당님 예상**(PL 준비된 안건만).
- **등급 수정(오버라이드)**: 상세에서 사람이 등급을 바꾼다. 원값·수정자·사유가 이력으로 남는다.
- **PPT 매핑 확인**: PPT를 올린 뒤 슬라이드가 Feature에 맞게 붙었는지 확인·수정. **매핑을 확정해야 PL 검사가 돌아간다**.
- **자료 보기**: 슬라이드 이미지를 좌우로 넘겨 보는 뷰어(PL 지적 사유 오버레이).

### 회의 (Meetings)
회의 일정을 짜고 SW담당 예측 정확도를 본다.

- **상단 2단**: 과제 일정(DVR 수정·마일스톤) / **SW 담당님 예상 적중률**(최근 5개 회의 종합 + 회의별 추이 막대).
- **회의일 지정**: 달력에서
  - **더블클릭** = 회의일 추가/삭제 (회의는 하루 한 번 = 날짜당 슬롯 1개).
  - **클릭** = 아래 안건 배정 보드에 그 회의일을 표시/숨김 (화면 상태일 뿐, 회의는 안 지워짐).
- **안건 배정**: 표시 중인 회의일 최대 5개 컬럼 + 미배정 컬럼. **드래그앤드롭**으로 안건을 옮기고, 드롭한 위치에 삽입된다. 컬럼의 ✕는 "보드에서 숨기기"(회의 삭제 아님).
- **회의 공지 메일 초안**: 각 회의일 컬럼의 "✉ 공지 메일 초안" 버튼 → 그 회의일 안건으로 메일 초안. **수신자**는 메일주소 열들(config `meeting_recipient_columns`)의 `.com`을 모아 `;`로 연결, **표**(인덱스·기능명·변경점·개발그룹·그룹장·개발담당자·EP/UX/CXI담당자)에는 담당자 **이름**이 들어가고 **본문 맨 끝**에 붙는다. 인사말·맺음말은 편집 가능하며 **서버에 팀 공유 저장**돼 다음에도 유지된다.
- 대면 회의 대상은 **P0·P1뿐** (P2는 서면, 단순 공유는 공유만).

### 일정 관리 (Schedule)
개발 일정이 임박한 항목을 챙기고 담당자에게 공지 메일을 보낸다.

- **임박 기준**(30/45/60/90일)을 골라 개발 일정이 그 안에 든 항목만 날짜별로 묶어 본다. 일정이 지났거나 지연된 항목은 호박색으로 강조된다. 표 열은 모든 날짜에서 같은 너비로 정렬된다.
- 날짜마다 **공지 메일 초안** 버튼:
  - **수신자 1줄** — 그 날짜 항목들의 담당자 메일(.com)을 중복 없이 `;`로 연결. [복사]로 메일 수신란에 바로 붙여넣기.
  - **인사말·맺음말** — 편집 가능한 문구. 한 번 다듬으면 **서버에 종류별로 저장**돼 누가 열어도 유지된다(팀 공유, "기본 문구로" 링크로 되돌리기). 날짜·시각은 제목·표에 들어가므로 문구엔 넣지 않는다. 표는 자동(수정 불가).
  - **아이템 표** — 인덱스·기능명·변경점·개발일정·지연사유. "본문 복사(표 포함)"로 메일에 붙이면 표가 유지된다.
- 담당자 메일은 엑셀의 개발 담당자 메일 열에서 자동으로 뽑는다(설정 → 스키마 매핑의 `dev_owner_mail`).
- **일정 지연 감지**: 갱신 인입 때 개발 일정이 이전보다 뒤로 밀린 항목은 ⏰ 지연 배지로 표시되고 알림 센터("일정 지연")에 뜬다. 일정을 미룬 기능은 **지연사유** 열에 사유를 적어야 하며, 비면 "미기재"로 경고된다.

### 추적 (Tracking)
회의에서 나온 결정을 실행·관리.

- **회의록 입력**: 회의록을 붙여넣으면 AI가 건별 결정(지원/보류/미지원)과 액션 아이템을 추출. **사람이 확인·수정한 뒤 확정**.
- **액션 아이템**: PLM으로 전송, 상태 추적(pending→sent→in_progress→done). 회사에서 `adapters/plm.py`를 넣으면 실제 PLM에 연결되고, 없으면 mock으로 단계만 진행된다.
- **후속 보고**: 보류 조건의 응답성을 AI가 판단해 후속 보고가 필요한 건을 표시.

### 인사이트 (Insight)
- 벤치마킹·동향 참고자료를 업로드(웹 검색 대신 파일로 제공).
- 버전의 주요 지향점·트렌드 정합성 리포트를 생성·열람.

### 질의 (Query)
- 채팅형으로 **전 버전 데이터를 자연어 검색**. 답변에 출처(버전·인덱스·날짜)가 붙는다.
- 기본은 2단계(키워드로 후보 추출 → 엔진 1회 호출), 심층 검색은 관리자만.

### 페르소나 (Personas)
- 프롬프트 13종(부문 4 + 종합 + PL + SW담당 + 보조 6)을 편집. 저장 시 자동 백업.
- **골든셋 관리**: 사람 정답 20~30건을 등록해 페르소나 수정 전후 일치율을 측정.
- SW담당 예측 적중률 추이.

### 설정 (Settings) — 관리자
- 기본 엔진(mock/gemini/claude)·배치 크기.
- 사용자·역할 관리.
- **엑셀 스키마 매핑**: 인입된 엑셀의 실제 열에서 선택 박스로 지정(항목 연결 → 열 용도 → 개발 완료 규칙, 저장 전 미리보기).
- 관리 열 다시 고르기.

### 알림 센터 / 로그·산출물
- 알림: 리뷰 완료, PL 탈락, 회의 임박, **개발일정 지연**, 작업 실패 등 이벤트 피드(유형별 필터).
- 로그: CLI 작업 로그, 보고 PPT 생성·다운로드.

## 4. 자주 하는 작업 레시피

| 하고 싶은 것 | 어떻게 |
|---|---|
| 새 버전 엑셀 올리기 | 현황판 → ① 인입 → 버전명·DVR 입력 → 엑셀 드래그 → 관리 열 선택 |
| 갱신본 반영 | 같은 버전으로 다시 업로드 → **트리거 열이 바뀐 행만** 재리뷰, 나머지는 유지 |
| 등급이 틀렸을 때 | 리뷰 보드 → 행 클릭 → 등급 수정(오버라이드) → 사유 입력 |
| 특정 조건만 보기 | 리뷰 보드 → 조건 필터 (예: 일정 리스크 있음) |
| 회의 잡기 | 회의 → 달력 더블클릭으로 회의일 추가 → 클릭해서 보드에 표시 → 안건 드래그 |
| 미지원된 건 보기 | 리뷰 보드 → 조건 필터 → "미지원 목록" (평소엔 숨겨짐) |
| 데모로 초기화 | `python scripts/gen_demo_data.py` (⚠ 실데이터 있으면 실행 금지) |

## 5. 용어 사전

**등급 (리뷰가 매김)** — "임원의 대면 판단이 얼마나 필요한가". 품질 평가가 아니다.
파급력(사용자 영향·전략 비중·영향 범위)과 대면 필요성(결정·쟁점 **또는** 공유·정렬) 두 축으로 정한다.
| 등급 | 뜻 |
|---|---|
| P0 최우선 대면 | 파급력 크고 결정 필요 — 회의 앞 순서 |
| P1 대면 | 쟁점이 제한적이거나, 이견은 없어도 공유·정렬 필요 |
| P2 서면보고 | 파급력 국소적 + 쟁점·공유 필요 모두 없음 |
| 단순 공유 | P2 중 하드룰로 골라낸 "공유만 하면 되는" 건 |
| 자료 보완 필요 | 변경점이 부실해 판정 불가 — AI 앞단에서 걸러짐 |

**결정 (회의에서 임원이 정함)** — 지원(이번 버전 반영) / 보류(유보·조건부·차기 검토) / 미지원(이번 버전 제외).
- **미지원**은 모든 통계·KPI 모수에서 빠지고, 다음 버전 재등록 이력 추적용으로만 남는다.

**단계 vs 조건** — 리뷰 보드 필터 참조(§3). 단계=진행(하나), 조건=골라보기(겹침).

**일정 리스크** — 개발 일정이 **DVR**(개발 검증 기준일)을 넘거나 일정이 미정이면 "있음". AI 없이 즉시 판정.

**DVR** — 과제 입력 시 넣는 개발 일정 리스크 판정 기준일. 회의 화면에서 수정 가능.
